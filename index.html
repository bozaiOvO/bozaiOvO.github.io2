01JQ的今生与未来。

jQuery是什么

  • jQuery是一个快速、简洁的JavaScript框架，是一个优秀的JavaScript代码库。jQuery设计的宗旨是“write Less，Do More”，即倡导写更少的代码，做更多的事情。它封装JavaScript常用的功能代码，提供一种简便的JavaScript设计模式，优化HTML文档操作、事件处理、动画设计和Ajax交互。 jQuery的核心特性可以总结为：具有独特的链式语法和短小清晰的多功能接口；具有高效灵活的css选择器，并且可对CSS选择器进行扩展；拥有便捷的插件扩展机制和丰富的插件。jQuery兼容各种主流浏览器，如IE 6.0+、FF 1.5+、Safari 2.0+、Opera 9.0+等。
  • 说人话就是，方便DOM操作，让你更简单的写js，也不需要担心兼容问题，JQ的核心就是抹平浏览器带来的差异！

jq的几个特性

  • 语法糖：链式调用、实用函数
  • 选择元素：基于各种标准选择符和库自定义的选择符，以及通过回调进行筛选
  • 操作DOM：创建和操作元素，乃至各种属性操作
  • 处理事件：ready事件及各种注册和触发事件的方法，乃至委托
  • 动画：基于animate的CSS属性动画，以及内置动画方法
  • Ajax：封装原生的XMLHttpRequest API，简化请求方法及回调处理

选择JQ的理由

  • 跨浏览器
  • 简单高效
  • 稳定可靠
  • 插件丰富

历史的选择，华丽的谢幕！

  • js的一代代更新，使原声的js就可以很方便的选取DOM节点，兼容性也在增强。（就是document.querySelector和document.querySelectorAll方法。）
  • 方便操作DOM元素的API，比如jq的addClass,removeClass,toggleClass,在原生js上也有了支持，api为classList
  • 动画方面，css3动画已经很成熟，取代绝大多数jq动画没问题，并且兼容以及性能更优化。
  • ajax操作，Jq的ajax操作，简化了相当多的代码，并且有良好的兼容性，但是现在这个优势也是逐渐式微，原声js的Fetch和axios都提供了更为强大的ajax使用能力，甚至于axios还有拦截器！
  • MVC框架（angular）以及MVVM框架（vue/react）的冲击，一代更比一代强！曾经的王者jq解决了浏览器差异的问题，我们学会jq就可以解决这些差异问题，但是随着浏览器标准的逐渐统一，jq的价值就逐渐式微，而新一代框架，例如vue，更让人专注数据层，vue提供了数据驱动以及虚拟DOM的思想，老的王者jq是让你方便的操作DOM，而新一代框架，根本就不需要让你操作DOM了，让你和数据打交道！！所以jq是被新一代的框架吊起来锤！（可维护性高，扩展性强，性能好）

百足之虫，死而不僵！

  • 纵然很多人叫嚣什么“JQ已死”，实际上jq仍然有一席之地！尤其是需要SEO优化的网站是不适合MVVM的！
  • 学习成本低，几乎没有学习成本，看看就会了！不用担心浪费时间，事实上有开发经验的人使用这个东西都是开箱即用，也不用刻意记一些api。
  • 内部系统和私人网站更适合VUE/angular/react。而to C产品，也就是面对消费者的产品，更多是用JQ。
  • 更多要看场景，而不是谁厉害就用谁！最厉害的未必是最好的，适合你的才是最棒的。
  • jq其实真正意义上并没有死，只是他的历史使命已经优秀的完成，他的优秀地方也被js标准所吸收。比如querySelector和classList等.....

*扩展

啥是MVVM？

  • MVVM分为Model、View、ViewModel三者。
  • Model 代表数据模型，数据和业务逻辑都在Model层中定义；
  • View 代表UI视图，负责数据的展示；
  • ViewModel 负责监听 Model 中数据的改变并且控制视图的更新，处理用户交互操作；
  • Model 和 View 并无直接关联，而是通过 ViewModel 来进行联系的，Model 和 ViewModel 之间有着双向数据绑定的联系。因此当 Model 中的数据改变时会触发 View 层的刷新，View 中由于用户交互操作而改变的数据也会在 Model 中同步。这种模式实现了 Model 和 View 的数据自动同步，因此开发者只需要专注对数据的维护操作即可，而不需要自己操作 dom。

03JQ的DOM操作

~

DOM创建节点以及节点属性。

  • 之前所学的都是直接定义好html结构，但是在实际开发中 某些场景下是需要后端动态的返回给你数据然后再插入的~
  • 创建元素节点document.createElement('div')
  • 设置属性：xx.setAttribute('class','xxx')
  • 添加Html:inndeHTML
  • 加入文档：appendChild
  • 这些东西最重要的问题就是浏览器兼容问题。

jq创建节点以及节点属性。

  • 直接使用$来把html结构表示出来$('<span>我一个快乐的SPAN</span>')

  • 他甚至可以把class等属性直接写里面。$('<span class='hi'>哈哈</span>')

  • 插入就是append和appendTo。此为append$body.append($('<span>嘿嘿</span>'))而appendTo就是反过来，需要插入的内容放在前面，把节点放在后面$('<span>李双蛋与王花花</span>').appendTo($('.div'))，这两个都是插入在内部。但是都是在内部最后追加！

  • 如果你想插入到内部并且在最前面的话 就是prepend()与prependTo.方法和append以及appendTo是一样的。

  • 插入在外部就是after和before，一个是在之后插入，一个是在之前插入。方法和append差不多。

  • 清空节点empty()，指的是清空所有的内部子节点。$('div').empty()

  • 删除节点并且移除自身以及全部的子节点$('div').remove()

  • 删除节点的同时也会把这个节点的事件给全部的清空，但是如果你想让该节点消失，但是这个节点的事件不丢失，你可以使用detach()，他会把这个节点的事件保存在内存里。

  • 克隆/添加/删除都有了，那就该替换了replaceWith。

04Jq遍历

  • jq是一个合集对象，如果你想快速寻找合集的第一季子元素，可以用children()方法。$('.box').children(),如果想要所有的子类，那就是$('.box').find('*')，当然如果说你要某一个$('.box').find('a')意为从box里拿到所有的a标签。
  • 能找孩子，就能找他爹。寻找父类元素就是$('.box').parent(),这个寻找的是直接父类。想要寻找全部的父类就是$('.box').parents(),全部的这个就没有什么参数了。
  • 下一个兄弟节点$('.box').next() ，上一个兄弟节点$('.box').prev(),全部的兄弟节点$('.box').siblings()
  • each方法，相当于循环函数，里面有两个参数。index和ele
